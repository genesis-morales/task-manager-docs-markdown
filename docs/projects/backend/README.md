# TaskManager Backend Project

## Overview

This directory will contain the backend application for the TaskManager project. The backend is an independent project that provides a RESTful API for managing tasks, projects, users, and integrations.

## Tech Stack

- **Framework**: FastAPI (Python)
- **Database**: PostgreSQL
- **ORM**: SQLAlchemy
- **Migrations**: Alembic
- **Authentication**: JWT (JSON Web Tokens)
- **Testing**: Pytest
- **Containerization**: Docker

## Architecture

The backend follows **Clean Architecture** with **Domain-Driven Design** (DDD) principles:

## Development Guidelines

### Clean Architecture Principles

1. **Domain Layer**: Core business logic, technology-agnostic
2. **Application Layer**: Use cases and orchestration
3. **Infrastructure Layer**: Technical implementations
4. **Presentation Layer**: API endpoints and request handling

### SOLID Principles

- **Single Responsibility**: Each class/module has one reason to change
- **Open/Closed**: Open for extension, closed for modification
- **Liskov Substitution**: Subtypes must be substitutable for base types
- **Interface Segregation**: Many specific interfaces > one general interface
- **Dependency Inversion**: Depend on abstractions, not concrete implementations

### Best Practices

- Use type hints for all function parameters and return values
- Write docstrings for all public functions and classes
- Follow PEP 8 style guide (enforced by black and flake8)
- Write tests for all business logic (aim for 80%+ coverage)
- Use dependency injection for better testability
- Keep domain logic independent of frameworks
- Use repository pattern for data access
- Implement proper error handling and logging

## API Documentation

API documentation is automatically generated by FastAPI and available at:

- **Swagger UI**: `http://localhost:8000/docs`
- **ReDoc**: `http://localhost:8000/redoc`
- **OpenAPI Spec**: `http://localhost:8000/openapi.json`
